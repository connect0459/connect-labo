# Technical Discussion with TC39

## メモ

- Explicit Resource Management Stage 3
  - new variable declartion style `using`
  - lint rule is changed
  - Node.js / Deno / Bun server side JS like this style
  - However Browser ...?
  - 要約
    - [using](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/using) の話から。
    - Chromeには実装されているので、もう少しサポートが増えれば正式にStage 4か？
    - 新しい宣言タイプの追加で、lintのルール変更が大変だった
      - async や await と同じく、変数にもなれるような語句を追加するのって意外に難しい（らしい）
- Module Harmony
  - Some sub specs are Stage 3
  - Others are Stage 0-2
  - Really interesting, look back current situation, re-organize module eco system.
- Record & Tuples (withdrawn)
  - 要約
    - 既存の仕様を変更する場合、変更コストは線形的ではなく指数的に増加する
    - `===`の変更はまさにそれ
- Decorators
  - same stage on long time
  - We would like to decorate functions, classes etc.
  - "use xxx" pragma is spread by 3rd party people.
  - How you feel about this?
  - 要約
    - フレームワークに機能追加として導入する場合、明示的ではあるのでそれ自体の設計は良いと思う
    - ディレクティブは言語だけが占有している組み込み機能ではないので、変な懸念はない。ユーザー側からそのような反応が来るのが面白いとは思う。
- Pipelined operator(Stage 2?)
  - `|>`
  - 要約
    - F#か他のパターンかで争っているのでまだ検討中
- 5 years
  - ECMAScriptのStageを上る段階と、ISOを定義する段階がある
